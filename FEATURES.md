## 优先级说明

- **P0**: 基础设施，无此则后续功能无法展开，必须最先做
- **P1**: 核心业务功能，日常运维高频使用
- **P2**: 重要但可延后，P0/P1 稳定后再推进
- **P3**: 锦上添花，有最好但不急

## P0 实施计划（当前阶段）

共 8 项，按依赖关系排序：

### 1. F1 前端重构

```
templates/

└── devices.html         # 设备台账页

├── js/
│   └── api.js           # 公共 fetch 封装
└── css/
    └── app.css          # 公共样式 + CSS 变量（明/暗主题）
```

- 侧边栏导航：总览 / 设备台账 / 告警中心 / ACL 管理（后续页面逐步加入）
- 移除浮动刷新按钮，改顶栏定时刷新
- 接口表格加搜索框和状态筛选
- 设备卡片按标签分组折叠

### 2. D1 多厂商抽象驱动
**依赖**: 无

```python
# 抽象基类
class DeviceDriver(ABC):
    @abstractmethod
    def collect_metrics(self, device) -> MetricsResult
    @abstractmethod
    def collect_interfaces(self, device) -> List[InterfaceData]
    @abstractmethod
    def execute_command(self, device, command) -> CommandResult

class H3CComwareDriver(DeviceDriver):   # SSH方式
class H3CComwareSNMPDriver(DeviceDriver):  # SNMP方式
```

### 3. D2 SNMP 采集 + D4 SSH 兜底
**依赖**: D1

- 用 `pysnmp` 采集标准 MIB (IF-MIB, IP-MIB 等)，替代 SSH 解析 CLI
- 采集失败时自动退回到 H3CComwareDriver (SSH)
- 新依赖：`pysnmp`、`pyasn1`

### 4. M1 告警规则引擎
**依赖**: D1, D2, F1（前端导航+告警页面）

- 每台设备可定义告警规则（CPU>80%、内存>90%、接口DOWN、温度>60°C）
- 采集完成后检查规则，触发告警写入 `alarms` 表
- 告警状态：active / acknowledged / resolved
- 前端告警中心页面：告警列表、筛选、确认

### 5. M5 接口流量实时图
**依赖**: D2, F1

- 接口列表页：搜索、状态筛选
- 接口详情：流量趋势图（in/out bps）、带宽利用率
- 后端新增接口速率字段，计算利用率

### 6. A1 设备台账
**依赖**: D1（驱动 refactor 后设备模型统一）

- 扩展 `devices` 表：序列号、采购日期、维保到期日、供应商、合同编号、资产编号、机房、机柜、角色
- 设备详情页 / 编辑表单
- 设备列表支持按标签/机房/角色筛选

### 7. C5 ACL 管理
**依赖**: D1, F1

- ACL 规则列表（从设备读取并展示）
- 新建/编辑/删除 ACL 规则
- 应用到接口（指定接口 + direction in/out）
- 后端：通过 SSH/NETCONF 下发配置

### 8. F1 收尾
- 暗色主题完善
- 响应式适配（平板可用）
