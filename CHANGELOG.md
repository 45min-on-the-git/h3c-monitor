# H3C Monitor 项目升级报告

## 项目概述

对开源的 H3C 网络设备监控项目进行了全面重构和功能扩展，从单一监控脚本升级为完整的网络运维管理平台。

## 升级前后对比

| 维度 | 原始版本 | 当前版本 |
|------|---------|---------|
| 采集方式 | SSH CLI 正则解析 | **SNMP 优先 → SSH 自动兜底** |
| 配置下发 | 无 | **NETCONF 优先 → SSH 兜底** |
| 页面数量 | 1 个单页 | **15 个页面** |
| API 路由 | 7 个 | **78 个** |
| 告警能力 | 无 | **规则引擎 + 自动触发/恢复 + 实时推送** |
| 设备管理 | 仅 IP 列表 | **完整台账（序列号/维保/机房/标签）** |
| 配置控制 | 无 | **端口/VLAN/ACL/路由 4 类操作** |
| 数据库表 | 3 个 | **11 个** |
| 单元测试 | 0 | **18 个** |
| 设备驱动 | 1 种（SSH） | **4 种（抽象基类 + SNMP + SSH + NETCONF）** |

## 新增功能详情

### 一、监视可视化

1. **SNMP + SSH 双通道采集** — SNMP 采集指标（CPU/内存/温度/接口流量），失败自动切 SSH，提升采集可靠性
2. **告警引擎** — 自定义阈值规则（CPU>80%、内存>90% 等），自动触发告警，指标恢复自动消除。顶栏铃铛 30 秒轮询，新告警弹窗提醒
3. **接口流量实时图** — 点击接口行查看流量时序曲线（bps），支持 24 小时内任意时段。带宽利用率颜色预警（绿<50% / 黄 50-80% / 红>80%）
4. **NOC 大屏** — 独立全屏深色页面，总设备数/在线数/告警数三大指标 + CPU/内存仪表条，10 秒自动刷新
5. **定时报表** — 选设备 + 时间范围（1/7/30 天），CPU/内存/温度的平均值和最大值聚合统计

### 二、设备生命周期管理

6. **设备台账** — 每设备 15+ 字段：序列号、采购日期、维保到期、供应商、合同编号、资产编号、机房、机柜、标签。支持多条件筛选和搜索
7. **IP 地址管理（IPAM）** — 子网规划和 IP 分配管理，支持利用率统计。左侧子网列表 → 右侧 IP 网格，标记 used/free/reserved
8. **VLAN 划分** — 选设备 → 实时读取 VLAN 列表 → 新建/删除 VLAN，通过 NETCONF/SSH 下发到设备
9. **配置备份** — SSH/NETCONF 拉取 running-config，存库带 hash。备份历史列表、内容查看、一键回滚
10. **变更审计** — 所有配置操作（端口启停、ACL 变更、VLAN 创建等）自动记录操作时间和详情，全程可追溯

### 三、设备配置控制

11. **端口管理** — 选设备 → 接口列表 → shutdown/undo 启停、修改描述、配置 VLAN（access/trunk 模式）、绑定 IP 地址
12. **ACL 管理** — 通过 SSH 实时读取设备 ACL → 新建/编辑/删除规则 → 下发到设备。兼容 H3C Comware V7 输出格式
13. **静态路由管理** — 读取设备静态路由表 → 添加/删除路由条目
14. **配置模板** — Jinja2 模板引擎，预写配置脚本 + 填变量 → 实时预览渲染结果，批量下发多台设备
15. **NETCONF 驱动** — 基于 ncclient 实现 H3C NETCONF 协议接入（端口 830），配置下发优先走 NETCONF，不可用时自动退 SSH

### 四、工程化提升

- **驱动抽象层** — 统一 DeviceDriver 接口，数据类定义（DeviceInfo/MetricsResult/InterfaceData），新增厂商驱动只需实现接口
- **前端模块化** — 侧边栏导航 + base.html 模板继承 + 13 个 JS 模块 + 明/暗双主题
- **18 个单元测试** — 覆盖 ACL 解析（7 例）、告警引擎（6 例）、数据库 CRUD（5 例）
- **数据库扩展** — 从 3 张表扩展到 11 张表（设备/指标/接口/告警规则/告警记录/IPAM/配置备份/配置模板/端口分配/审计/VLAN）

## 技术栈

Python 3.10+ / FastAPI + SQLite + APScheduler + Jinja2 /
Netmiko (SSH) + pysnmp 7.x (SNMP) + ncclient (NETCONF) /
Bootstrap 5 + Chart.js + 原生 JavaScript（零前端构建工具）
